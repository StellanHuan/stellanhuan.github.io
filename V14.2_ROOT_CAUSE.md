# V14.2 Root Cause Analysis

## CDN CSS Cascade (verified live)

Desktop rules (outside @media):
  .masonry{display:grid;grid-template-columns:2fr 1fr 1fr}  ← line ~116
  .about-section{display:flex;gap:4rem}                      ← line ~171
  .block-head{display:flex;justify-content:space-between}    ← line ~108

Mobile rules (inside @media(max-width:768px)):
  .masonry{display:flex!important}      ← ONLY when viewport ≤768px
  .about-section{flex-direction:column!important}
  .block-head{flex-direction:column}

## Cascade is CORRECT
Desktop rules apply at >768px. Mobile !important only at ≤768px.
No CSS leak outside @media. Verified on CDN.

## Most likely cause of 瓛瓛''s issue
1. Browser width ≤ 768px (iPad portrait, unmaximized window)
2. Browser zoom level > 100% reduces effective viewport
3. CDN still serving cached pre-fix version (wait 2-3 min)
4. DevTools device mode accidentally left on mobile preset
